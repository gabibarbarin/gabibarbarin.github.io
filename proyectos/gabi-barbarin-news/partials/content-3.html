¿HTML qué es y para qué sirve?

HTML es el lenguaje de las páginas webs, todas las páginas webs están hechas en HTML

Pero HTML al contrario de lo que mucha gente piensa no es un lenguaje de programación, es un lenguaje de marcas o de marcado.

HTML está pensado para estructurar la información que se le da al navegador para visualizar las páginas web, de forma que este entienda que es cada cosa.
Un documento HTML tiene un aspecto como este.

El HTML como buen lenguaje de marcado, se basa en un sistema de etiquetas. Estas etiquetas se usan para que el navegador comprenda que clase de información le estamos dando. Por ejemplo, tenemos etiquetas para indicar títulos, toda la información que metamos en esas etiquetas, el navegador sabrá que son títulos. O por ejemplo imágenes, tenemos etiquetas para imágenes, de esta forma el navegador sabrá que es una imagen, y la pintará como tal.

¿Cómo se indica que es un etiqueta?

Las etiquetas en HTML comienzan con un símbolo de menor que, y terminan con uno de mayor que. Entre estos dos símbolos va el nombre de la etiqueta.

Por ejemplo, tenemos una etiqueta para indicar un párrafo, el nombre de esta etiqueta es p. Después de cerrar esta etiqueta, escribiríamos el párrafo de texto, y cuando terminas al final volvemos a poner la etiqueta, pero esta vez con una barra diagonal, esto indica que es el cierre de la etiqueta, que lo que hay entre la primera etiqueta sin la barra diagonal, y la que la tiene, es la información del párrafo.

Estructura de un archivo HTML

Un archivo de HTML no es más que un archivo de texto plano donde ponemos todas estas etiquetas e información.

Con su apertura y con su cierre, y dentro de esta etiqueta, están todas las demás.

Tenemos dos partes muy importantes, la primera es el:

Head, o cabecera

Donde van etiquetas especiales con información que no la ve el usuario, como puede ser el título de la página, la descripción, o las vinculaciones a los archivos CSS, donde van los estilos, y JavaScript, donde va la lógica de la web.

Segunda parte, la etiqueta body

Aquí tenemos lo que se ve en el navegador, toda la información visible al usuario. Como pueden ser títulos, párrafos, imágenes, cualquier cosa que se te ocurra que sea visible para el usuario.

Oye, esa etiqueta img que he visto, no tiene dos partes, como las otras.

Cierto, hay etiquetas que no se las rellena con información entre la etiqueta de apertura y la de cierre, como la de img, que como vemos tiene lo que llamamos un atributo llamado src (de source, fuente de la imagen), donde va la URL o la ruta donde tenemos la imagen que queremos que muestre.Y como vemos en este tipo de etiquetas, el cierre está al final de la etiqueta, al poner la barra diagonal al final de la etiqueta estamos indicando que ya se cierra ahí, y que no necesita de una segunda parte de cierre.
